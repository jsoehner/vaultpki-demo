# Porting Workflows to Non-Java Projects

To port your custom `nightly-dependency-update.yml` and `security-scan.yml` workflows to other languages (like Node.js, Python, Go, etc.) while keeping the same logic—such as pre-verifying dependencies before creating PRs and creating non-blocking GitHub issues for security findings—you need to replace the Java/Maven specific steps with their language-specific equivalents.

Here are the generalized templates.

## 1. Generic Nightly Dependency Update

This workflow updates dependencies, runs tests to verify the updates didn't break anything, and creates a PR (falling back to a manual issue if permissions are blocked).

```yaml
name: Nightly Dependency Update (Generic)

on:
  schedule:
    - cron: '0 2 * * *'
  workflow_dispatch:

permissions:
  contents: write
  pull-requests: write
  issues: write

jobs:
  update-dependencies:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Configure Git for automated commits
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"

      # --- START LANGUAGE SPECIFIC SETUP ---
      # Example for Node.js:
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      # --- END LANGUAGE SPECIFIC SETUP ---

      # --- START DEPENDENCY UPDATE ---
      # Example for Node.js:
      - name: Update dependencies
        run: |
          npm update --save
      # --- END DEPENDENCY UPDATE ---

      # --- START BUILD AND VERIFY ---
      # Example for Node.js:
      - name: Build and Verify
        run: |
          npm ci
          npm test
      # --- END BUILD AND VERIFY ---

      - name: Clean generated artifacts from branch
        run: |
          # Remove any build artifacts generated during testing
          # Example for Node.js:
          git rm -r --cached dist coverage node_modules || true
          git status --short

      - name: Create Pull Request
        id: create-pr
        uses: peter-evans/create-pull-request@v6
        continue-on-error: true
        env:
          PR_TOKEN: ${{ secrets.PERSONAL_ACCESS_TOKEN }}
        with:
          token: ${{ env.PR_TOKEN || github.token }}
          commit-message: "chore(deps): update project dependencies"
          title: "Automated Dependency Update"
          body: |
            This PR contains automated dependency updates generated by the nightly workflow.
            The dependencies were updated, and a successful build/test was triggered.
          branch: "automated/dependency-updates"
          delete-branch: true
          assignees: "jsoehner"
          reviewers: "jsoehner"
          labels: "automated-dependency-update"

      - name: Notify manual PR approval needed
        if: ${{ always() && steps.create-pr.outcome == 'failure' }}
        uses: actions/github-script@v7
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
          script: |
            const owner = context.repo.owner;
            const repo = context.repo.repo;
            await github.rest.issues.create({
              owner,
              repo,
              title: 'Automated dependency branch ready for manual PR',
              body: `The automated dependency branch passed build verification but PR creation was blocked by repository policy.\n\nBranch: https://github.com/${owner}/${repo}/tree/automated/dependency-updates\n\nRequested reviewer: @jsoehner`,
              assignees: ['jsoehner'],
              labels: ['dependency-update']
            });
```

## 2. Generic Security Scan

This workflow retains **Gitleaks** (secrets) and **Trivy** (SCA/Containers) as they are completely language-agnostic. 

SpotBugs (Java SAST) is replaced with **Semgrep**, a powerful open-source, language-agnostic SAST scanner. OWASP Dependency-Check is usually redundant with Trivy for non-Java projects, so it is omitted to speed up the scan (Trivy scans package managers like npm, pip, go mod natively).

```yaml
name: Security Scan (Generic)

on:
  push:
    branches: [ "main", "master" ]
  pull_request:
    branches: [ "main", "master" ]
  schedule:
    - cron: '0 0 * * *'

permissions:
  contents: read
  issues: write
  actions: write

jobs:
  security-assessment:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Secret Scanning (Gitleaks)
        id: gitleaks
        uses: gitleaks/gitleaks-action@v2
        continue-on-error: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      # --- SAST Scanning ---
      # Semgrep supports Python, JS/TS, Go, Java, C#, Ruby, PHP, etc.
      - name: SAST Scanning (Semgrep)
        id: semgrep
        continue-on-error: true
        run: |
          pip install semgrep
          semgrep scan --config auto --json -o semgrep-results.json

      # --- SCA & Container Scanning ---
      - name: Container & FS Scanning (Trivy)
        id: trivy
        uses: aquasecurity/trivy-action@master
        continue-on-error: true
        env:
          TRIVY_DB_REPOSITORY: ghcr.io/aquasecurity/trivy-db:2
        with:
          scan-type: 'fs'
          scan-ref: '.'
          format: 'json'
          output: 'trivy-results.json'
          exit-code: '1'
          severity: 'CRITICAL,HIGH'

      - name: Parse scan reports
        if: ${{ always() }}
        run: node .github/scripts/parse-findings.js

      - name: Notify on security findings
        if: ${{ always() && (steps.gitleaks.outcome == 'failure' || steps.semgrep.outcome == 'failure' || steps.trivy.outcome == 'failure') }}
        uses: actions/github-script@v7
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
          script: |
            const fs = require('fs');
            const owner = context.repo.owner;
            const repo = context.repo.repo;
            const runId = context.runId;
            
            let body = `The security scan workflow detected potential findings in the codebase.\n\n`;
            body += `Please review the workflow run logs for details: https://github.com/${owner}/${repo}/actions/runs/${runId}\n\n`;
            
            if (fs.existsSync('findings-table.md')) {
              body += fs.readFileSync('findings-table.md', 'utf8');
            } else {
              body += `**Failed Scan Steps:**\n`;
              if ('${{ steps.gitleaks.outcome }}' === 'failure') body += `- Secret Scanning (Gitleaks)\n`;
              if ('${{ steps.semgrep.outcome }}' === 'failure') body += `- SAST Scanning (Semgrep)\n`;
              if ('${{ steps.trivy.outcome }}' === 'failure') body += `- Container & FS Scanning (Trivy)\n`;
            }
            
            await github.rest.issues.create({
              owner,
              repo,
              title: 'Security Scan: Findings Detected',
              body: body,
              assignees: ['jsoehner'],
              labels: ['security', 'automated-scan']
            });

            // Trigger Automated Dependency Update
            console.log('Triggering automated dependency update workflow...');
            try {
              await github.rest.actions.createWorkflowDispatch({
                owner,
                repo,
                workflow_id: 'nightly-dependency-update.yml',
                ref: context.ref || 'main'
              });
              console.log('Successfully triggered dependency update workflow.');
            } catch (error) {
              console.error('Failed to trigger dependency update workflow:', error);
            }
```

### Note on `parse-findings.js`

If you implement the generic Security Scan, you will also need to update `.github/scripts/parse-findings.js` in the target repositories to support parsing **Semgrep** output (since SpotBugs and OWASP Dependency-Check were removed).

Here is the snippet to parse Semgrep's JSON output:
```javascript
let semgrepFindings = [];
if (fs.existsSync('semgrep-results.json')) {
  try {
    const raw = fs.readFileSync('semgrep-results.json', 'utf8');
    const data = JSON.parse(raw);
    if (data.results && Array.isArray(data.results)) {
      for (const result of data.results) {
        semgrepFindings.push({
          tool: 'Semgrep',
          file: `${result.path}:${result.start.line}`,
          description: result.extra.message,
          severity: result.extra.severity || 'HIGH'
        });
      }
    }
  } catch (e) {
    console.error('Error parsing Semgrep report:', e);
  }
}
// Add semgrepFindings to the allFindings array
```
